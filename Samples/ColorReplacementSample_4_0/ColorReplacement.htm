Color Replacement Sample

This sample demonstrates how to render parts of an object with a user-defined color.

Sample Overview

When it is the Red Team versus the Blue Team or when you want to drive a car painted with your favorite color, you either need to create a texture for each color or to perform color replacement programmatically. This sample implements color replacement using an artist-defined alpha map that acts similar to a spray paint stencil. The stencil enables you to have soldiers with any color armor you choose without tinting their faces the same color.

How the Sample Works

A conventional effect transforms a model, then shades it based on the texture RGB colors and the intensity of light hitting a given pixel. This effect goes a step further, combining the normal lighted color with a tinting factor based on an effect parameter and the alpha channel value of the texture.

This sample loads a model file, Car.x, during initialization. Car.x is hard coded to use ReplaceColor.fx as its effect. In DrawModel, three effect parameters are set before each mesh is drawn: a World/View/Projection matrix for transforming the vertices, a separate World matrix for lighting calculations, and a Color for tinting the final model.

The vertex shader for ReplaceColor.fx transforms each vertex into World/View/Projection space, and calculates the amount and color of light impacting that vertex (for more information on vertex shading, see Shader Series 1). This information, along with the texture coordinate of the vertex, is passed to the GPU.

Once that is done, the ReplaceColor.fx pixel shader is invoked for each pixel on the car model. The pixel shader inputs are the same as the vertex shader's outputs; however, they are adjusted by the GPU for each pixel's location. The pixel shader uses the diffuse color (lightDiffuse) and the texture coordinate (texCoord) from the vertex shader. It also accesses the texture in sampler 0, defined at the top of the effect as DiffuseTexture. This texture is referenced by the model file, and it is inserted into sampler 0 automatically by XNA. The RGB channels of this texture contain normal color information for a race car model. The alpha channel, however, contains a value that indicates what parts of the object can be tinted and by how much. An alpha value of zero indicates that the base color should be unchanged, while an alpha value of one indicates that the target color should completely replace the base color.

The first step inside the pixel shader is to use the tex2D command to sample the car's texture at the current pixel location, using the texture coordinates provided by the GPU. Next, the base color (from the texture) and the target color (from the effect parameter) are linearly interpolated based on the alpha value stored in the texture. Finally, this new model color is multiplied by the lighting color from the GPU, after which it returns as the pixel color.

Adapting the sample to your game

If you write your own Effects for rendering 3D objects in your game, you can create your own object textures. Use the alpha channel to control which parts of the object are tinted, and use ReplaceColor.fx as a base for your own vertex and pixel shader.
